create

최소지원버전
1.0.0.0
지원환경
멀티 모듈, IE 모듈 모두 지원
형식
dx5.create({ /* 생성 옵션 */ });
파라미터

create 함수는 다음의 속성을 갖는 파라미터 객체를 필요로 합니다.

이름 지원 설명
mode 공통

"multi" 또는 "ie" 값을 갖습니다.

"multi"이면 DEXTUploadX5 멀티 모듈(HTML5, SVG)이 생성됩니다.

"ie"이면 DEXTUploadX5 IE 모듈(ActiveX)이 생성됩니다.
style 공통

컴포넌트의 기본 GUI 모양을 결정합니다.

멀티 모듈에서는 "list", "tile", "single" 값을 가지며, 기본값은 "list"입니다.

IE 모듈에서는 "report", "report_noheader", "report_simple" 값을 가지며, 기본값은 "report"입니다.
progressType 공통

컴포넌트의 업로드/다운로드 진행창 GUI 모양을 결정합니다. '0', '1', '2', '3' 값을 가지며, 기본값은 '0'입니다.

멀티 모듈을 사용하여 ORAF 방식으로 업로드를 하는 경우, 파일 별 업로드 상태를 파악할 수 없는 관계로 '0'번 또는 '3'번 진행창만 사용이 가능합니다.

'3'번 진행창 형식은 파일 다운로드 작업을 수행할 때, 파일의 크기를 미리 알 수 없는 경우에 사용하도록 2.3.0.0 버전부터 추가되었습니다.
id 공통

[필수] DEXTUploadX5 컴포넌트의 아이디입니다.

컴포넌트가 생성될 때, 사용되는 object 엘리먼트의 id 속성 값이며, dx5 객체는 이 아이디를 가지고 컴포넌트를 구분합니다.
parentId 공통

[필수] DEXTUploadX5 컴포넌트가 생성될 부모 엘리먼트 아이디입니다.

컴포넌트가 생성될 때, 사용되는 object 엘리먼트를 포함할 컨테이너 엘리먼트의 id 속성 값입니다.
lang 공통

DEXTUploadX5 컴포넌트가 지원해야할 언어입니다.

"en", "ko", "ja", "zh", ...와 같은 값을 사용합니다. 기본값은 브라우저 언어셋을 따릅니다.

제품은 기본적으로 한국어 리소스만을 지원하기 때문에, 한국어 외의 다른 언어셋을 사용하려면 해당 언어로 작성된 dextuploadx5-svg-message-{언어셋}.txt 또는 dextuploadx5-ax-message-{언어셋}.xml이 필요합니다.

width 공통

컴포넌트의 가로 크기입니다.

CSS 형식을 사용하며, 기본값은 "100%"입니다.

컴포넌트의 가로 크기는 width 속성을 사용하지 않고 컨테이너의 가로 크기에 의존하는 것이 좋습니다.
height 공통

컴포넌트의 세로 크기입니다.

CSS 형식을 사용하며, 기본값은 "100%"입니다.

컴포넌트의 세로 크기는 height 속성을 사용하지 않고 컨테이너의 세로 크기에 의존하는 것이 좋습니다.

DEXTUploadX5 컴포넌트의 웹 위치입니다.

형식) 설명1|세미콜론으로 구분된 확장자 목록1|설명2|세미콜론으로 구분된 확장자 목록2||

ex) 이미지파일(jpg,jpeg,gif,bmp)|*.jpg;*.jpeg;*.gif;*.bmp;|문서파일(txt)|*.txt;*||

object.setExtensionFilter()

파일 업로드 혹은 다운로드 시 초당 전송 데이터량을 설정합니다.

설명

dx5는 DEXTUploadX5 컴포넌트를 사용하기 위해, 컴포넌트를 생성하고 얻는 작업을 합니다.

<script src="../dx5/dextuploadx5-configuration.js"></script>
<script src="../dx5/dextuploadx5.js"></script>

dx5는 DEXTUploadX5의 줄임말로, DEXTUploadX5의 컴포넌트를 사용하기 위한 스크립트 환경의 대표 구성 요소입니다.

dx5는 dextuploadx5.js 파일을 선언하면, 페이지 스크립트 로드 시점에 생성이 되는 자바스크립트 객체이므로 명시적으로 선언할 필요가 없습니다.

dextuploadx5.js 스크립트 파일은 DEXTUploadX5가 사용되는 모든 페이지에 포함되어야 하며 dextuploadx5-configuration.js 파일 다음에 선언되어야 합니다.

컴포넌트를 생성하기 위해서는 dx5.create 함수를 사용합니다.

<div id="dext5-container" style="width:500px; height:300px;"><!-- 컴포넌트를 담는 컨테이너 요소 --></div>
<button id="btn-add-files" type="button">파일 추가</button>
<script>
  dx5.create({
    // mode를 "multi" 또는 "ie"로 설정하며, id 속성으로 컴포넌트 DOM 아이디를 부여합니다.
    // 이 DOM 아이디는 컴포넌트를 구분 짓는 아이디이기도 합니다.
    // parentId는 컨테이너 아이디를 지정합니다.
    mode: "multi", id: "dext5", parentId: "dext5-container",
    // 컴포넌트는 버튼을 갖고 있지 않으므로 버튼 요소를 직접 만들어주어야 합니다.
    // button, input[type=button] 요소를 만들고 대상 아이디를 지정해주면 파일 추가 기능이 자동으로 연결됩니다.
    btnFile: "btn-add-files"
  });
</script>
<script>
  function onDX5Error(id, code, msg) {
    // 컴포넌트 사용 중에 오류가 발생하면 호출됩니다.
  }

  function onDX5Created(id) {
    // dx5.create 함수를 사용하여 컴포넌트가 만들어지면 onDX5Create 콜백 함수가 호출됩니다.
    // 컴포넌트가 생성되면 dx5.get 함수를 사용하여 컴포넌트를 얻을 수 있습니다.
    var dx = dx5.get(id);
  }
</script>
