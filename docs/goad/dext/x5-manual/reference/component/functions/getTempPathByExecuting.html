getTempPathByExecuting

최소지원버전
1.0.0.0
지원환경
IE 모듈 지원
형식
path = object.getTempPathByExecuting();
파라미터
반환 값
파일 복사본이 생성되는 경로
설명

파일 복사본이 생성되는 경로를 반환합니다.

필요에 따라 파일 원본이 아닌 파일의 복사본을 열어야 할 때가 있습니다.

// 임시 파일이 생성될 위치를 설정합니다.
// <X5TEMP>: DEXTUploadX5에서 정한 임시 위치
// <WINTEMP>: Windows 임시 폴더
// 드라이브 문자로 시작하는 경로: 대상 경로
dx.setTempPathByExecuting("c:\\temp");

// 4 번째 가상 파일을 복사본을 생성하여 로컬 프로그램을 실행시킵니다.

// 두 번째 파라미터가 true이면 원본 경로를 대상으로 합니다. 원본을 대상으로 하려면 대상 가상 파일이 다운로드가 되어 있어야 합니다.
// 그러나 다운로드가 안되어 있다면, 원본 경로 실행을 무시하고, 무조건 복사본을 실행하게 됩니다.
// 이때 복사본의 위치는 setTempPathByExecuting 함수를 사용하여 설정된 위치가 됩니다.
// 세 번째 파라미터는 프로그램을 실행할 때, 프로그램의 두 번째 파라미터로 전달될 문자열입니다.
dx.executeItem(3, false, "");
