decrypt

최소지원버전
1.0.0.0
지원환경
IE 모듈 지원
형식
object.decrypt(flag);
파라미터
이름 형식 설명
flag String "AUTO", "SELECTED", "CHECKED"
반환 값
설명

로컬 자원을 복호화합니다.

암호화된 파일을 복호화할 수 있습니다.

복호화 기능은, DEXTUploadX5로 암호화된 파일 만을 대상으로 하며, 복호화를 하려면 먼저 다운로드가 완료된 상태이어야 합니다.

복호화하는 과정에서 복호화 키를 질의합니다. DEXTUploadX5는 파일마다 키를 개별적으로 묻지 않습니다.

파라미터 값에 따라 복호화할 대상을 달리합니다.

  • "AUTO": 모든 파일 항목을 복호화합니다.
  • "SELECTED": 선택된 파일 항목을 복호화합니다.
  • "CHECKED": 체크된 파일 항목을 복호화합니다.
// 복호화 다이얼로그 타이틀을 설정합니다.
dx.setDecryptionDialogTitle("암호화 중...");

// 암호화된 파일을 가지고 복호화할 위치를 설정합니다.
// <X5TEMP>: DEXTUploadX5에서 정한 임시 위치
// <WINTEMP>: Windows 임시 폴더
// 드라이브 문자로 시작하는 경로: 대상 경로
dx.setDecryptionSavePath("c:\\temp");

// 복호화된 파일 생성되었는지 확인하기 위해, 탐색기 창을 엽니다.
dx.setDecryptionEnsuringFile(true);

// 복호화 작업을 시작합니다.
// 'AUTO', 'SELECTED', 'CHECKED' 옵션이 있습니다.
dx.decrypt("AUTO");

암호화된 파일을 업로드할 경우, 서버 측에서 복호화할 수 있는 기능을 따로 제공하지 않습니다.

DEXTUploadX5 컴포넌트를 사용하여 암호화된 파일은, 컴포넌트를 통해서만 복호화할 수 있도록 구현이 된 것은 의도된 정책입니다.
