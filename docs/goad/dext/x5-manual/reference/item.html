Item Object

최소지원버전
1.0.0.0
지원환경
멀티 모듈, IE 모듈 모두 지원
설명

항목(item)이란 DEXTUploadX5에서 다루어지는 자원을 의미합니다.

항목은 로컬 파일, 로컬 폴더, 가상 파일로 구분됩니다.

컴포넌트는 스크립트 상에서 항목(또는 다수)을 얻기위해 여러 가지 함수를 제공합니다.

// 여섯 번째 항목을 얻습니다.
var item = dx.getItemByIndex(5);
// 항목의 이름을 출력합니다.
console.log(item.name);

항목은 항목의 정보를 가지고 있는 여러 속성들을 노출하고 있습니다.

이름 지원 형식 설명
controlId 공통 String 항목이 생성된 컴포넌트의 아이디
id 공통 String 항목의 고유 아이디
type 공통 String

"FILE": 로컬 파일

"FOLDER": 로컬 폴더 (IE 모듈에서만 지원)

"VIRTUAL": 가상 파일

"DVIRTUAL": 삭제된 가상 파일
name 공통 String 항목의 이름, 파일 또는 폴더명
ext 공통 String 항목의 확장자
vindex 공통 String

항목의 가상 인덱스

가상 파일의 경우, 항목은 가상 인덱스를 설정합니다.

항목의 고유 아이디는 컴포넌트 내부에서 결정되지만, 가상 인덱스는 직접 지정해줘야 합니다.

일반적으로 데이터베이스에서 사용되는 키(key) 값이 됩니다.

ex) vindex: "1234567890"
size 공통 Number

항목의 크기(바이트 단위)

파일을 다운로드할 때, 올바른 진행 정보를 보여주려면 파일의 크기를 알아야 합니다.

가상 파일만 다운로드의 대상이 되므로, 가상 파일을 지정할 때에는 가능한 서버에 존재하는 대상 파일의 크기를 정확히 입력해야 합니다.
mdate 공통 Date 로컬 항목의 수정 일자
lock 공통 Boolean

항목의 잠금 상태이면 true, 아니면 false

항목을 잠그면 삭제를 할 수 없습니다.
status 공통 String

항목의 상태

"WAIT": 업로드 전, "DONE": 업로드 후
checked 멀티 Boolean 항목의 체크 상태
openUrl 멀티 String

항목의 열기 경로

가상 파일은 다운로드와 열기 기능이 지원됩니다.

openUrl은 열기를 수행할 대상이 있는 웹 주소로써, 스키마(http, https)가 포함된 전체 URL이어야 합니다.

열기란 브라우저가 직접 열 수 있는 파일만 대상이 되며, 그 외의 파일들은 다운로드가 됩니다.

브라우저가 열 수 있는 파일일지라도 attachement 형식으로 전달이 되면, 다운로드가 되어 버립니다. openUrl은 반드시 inline 형식으로 전달받도록 서비스를 구성해야 합니다.

downUrl 멀티 String

항목의 다운로드 경로

가상 파일은 다운로드와 열기 기능이 지원됩니다.

downUrl은 다운로드를 수행할 대상이 있는 웹 주소로써, 스키마(http, https)가 포함된 전체 URL이어야 합니다.

HTML5를 지원하는 모든 브라우저가 download 기능을 아직 모두다 지원하지 못하기 때문에, 대상 주소는 서버 컴포넌트를 사용하여 attachement 형식으로 전달이 되어야만 합니다.

path IE String

항목의 로컬 경로

대상이 로컬 자원(파일, 폴더)일 때, 드라이브 문자열(C:\, D:\, E:\)로 시작하는 대상의 로컬 경로를 가리킵니다.
middlePath 공통 String

항목의 폴더 경로

대상이 로컬 자원(파일, 폴더)일 때, 자원의 부모 폴더를 가리킵니다.

부모 폴더 경로는 드라이브부터 시작하는 것이 아니라, 폴더 선택 다이얼로그를 통해서 선택된 폴더의 하위 경로를 의미합니다.

멀티 모듈은 크롬 브라우저를 사용한 경우만 지원됩니다.
url 공통 String

항목의 웹 경로

가상 파일은 다운로드와 열기 기능이 지원됩니다.

url은 다운로드 또는 열기를 수행할 대상이 있는 웹 주소로써, 스키마(http, https)가 포함된 전체 URL이어야 합니다.

멀티 모듈은 1.4.1.0 버전부터 url 속성을 지원하며, 이 속성은 downUrl과 동일하게 다루어집니다. (멀티 모듈에서 열기 기능은 url이 아닌 기존의 openUrl을 그대로 사용합니다.)
cdate IE Date 로컬 항목의 생성 일자
imageWidth IE Number 이미지 파일의 경우 가로 크기
imageHeight IE Number 이미지 파일의 경우 세로 크기
exif IE String

EXIF를 지원하는 이미지 파일의 경우 대상이 갖는 EXIF 정보

EXIF 정보를 추출하도록 설정되어 있을 때만 사용할 수 있습니다.
eventUriStart 공통 String

버전 2.0.0.0부터 지원하는 속성입니다.

HD 응용프로그램을 사용하여 파일 다운로드를 이용할 때 사용되는 이벤트 주소를 나타냅니다.

이벤트 주소는 웹 주소로써, 스키마(http, https)가 포함된 전체 URL이어야 합니다.

HD 응용프로그램은 항목을 다운로드할 때, 이 속성이 설정되어 있으면 다운로드를 수행하기 전에 해당 주소로 GET 요청을 보냅니다.
eventUriStop 공통 String

버전 2.0.0.0부터 지원하는 속성입니다.

HD 응용프로그램을 사용하여 파일 다운로드를 이용할 때 사용되는 이벤트 주소를 나타냅니다.

이벤트 주소는 웹 주소로써, 스키마(http, https)가 포함된 전체 URL이어야 합니다.

HD 응용프로그램은 항목을 다운로드할 때, 이 속성이 설정되어 있으면 다운로드가 중지되면 해당 주소로 GET 요청을 보냅니다. (오류로 인한 중지가 아니라, 사용자 액션에 의한 중지를 의미합니다.)
eventUriEnd 공통 String

버전 2.0.0.0부터 지원하는 속성입니다.

HD 응용프로그램을 사용하여 파일 다운로드를 이용할 때 사용되는 이벤트 주소를 나타냅니다.

이벤트 주소는 웹 주소로써, 스키마(http, https)가 포함된 전체 URL이어야 합니다.

HD 응용프로그램은 항목을 다운로드할 때, 이 속성이 설정되어 있으면 다운로드가 완료되면 해당 주소로 GET 요청을 보냅니다.
chunkSize 공통 Number

버전 2.0.0.0부터 지원하는 속성입니다.

HD 응용프로그램을 사용하여 파일 다운로드를 이용할 때 파일이 분할 되는 크기를 나타냅니다.

HD 응용프로그램을 파일을 다운로드할 때, 한 요청으로 파일을 한 번에 다운로드하지 않고, 주어진 크기만큼 나눠서 다운로드를 수행합니다. 파일을 나눠서 다운로드를 하면 파일 다운로드 시간은 조금 더 오래 걸릴 수 있지만, 서버를 좀 더 안정적으로 운영할 수 있습니다. 다운로드할 파일의 크기가 크다면 해당 파일이 다운로드 완료될 때까지 대역폭을 붙잡고 있기 때문에, 성능상 문제가 발생할 수 있습니다. 이런 경우에 파일을 분할하면, 서버로의 다운로드 요청이 나눠지기 때문에, 일부 사용자가 서버 리소스를 장시간 점유하는 현상이 줄어들게 됩니다.

지정이 되어 있지 않으면 -1 값으로 설정이 되며, -1 또는 0인 경우에는 파일을 분할하지 않고 하나의 요청으로 다 받습니다. 더불어 분할 크기는 MB 단위로만 지정해야 합니다.
hdTitle 공통 String

버전 2.1.2.0부터 지원하는 속성입니다.

HD 응용프로그램을 사용하여 파일 다운로드를 이용할 때 윈도우 타이틀에 출력되는 텍스트를 나타냅니다.

파일이 다운로드 될 때, HD 응용프로그램의 윈도우 타이틀 영역에 텍스트와 파일 이름이 출력되는데, 지정된 값이 없다면 제품 이름과 버전이 출력됩니다.

컴포넌트에서 제공하는 함수를 사용하여 얻어진 항목의 대부분 복사본이므로, 항목의 속성 값을 변경했다고 하더라도 반영되지 않습니다.
