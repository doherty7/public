버전 업데이트

DEXTUploadX5 업데이트 버전은 덱스트솔루션 사이트(http://www.dextsolution.com)에 올라옵니다.

만일 새로운 버전으로 제품을 교체해야 한다면, 사이트에 올라온 평가판 버전을 다운로드하여 업데이트할 수 있습니다.

업데이트 기본 절차

기존 제품 리소스가 있는 경로에서 dextuploadx5-configuration.js 파일을 백업합니다.

만약 언어 리소스 파일을 수정했거나, 새로 추가된 리소스 파일이 있다면 대상 파일을 모두 백업합니다. (변경된 사항이 없다면 언어 리소스 파일을 백업하지 않아도 상관없습니다.)

덱스트솔루션 사이트(http://www.dextsolution.com)에서 평가판을 다운로드 받습니다.

배포 파일(dextuploadx5-{version}.zip)을 압축 해지합니다.

압축을 풀었을 때 생성되는 dx5 디렉터리 안의 모든 리소스를 복사하여 기존 리소스를 교체(덮어쓰기 또는 지우고 복사)합니다.

dextuploadx5-configuration.js 파일을 열어서 백업했던 설정 파일에 작성된 값을 버전 속성을 제외하고 모두 복원합니다.

// dextuploadx5-configuraiton.js 파일 내용입니다.
win.dextuploadx5Configuration = {	
  // verion 속성을 제외하고 productPath, ieDownloadURL, hdDownloadURL, h32UpdateURL, h64UpdateURL 모두 수정해야 합니다.
  ...
};

백업했던 언어 리소스 파일(txt, xml) 내용 중에 직접 추가하거나 수정했던 부분을 새로운 리소스 파일에 그대로 적용합니다.

1.4.1.0 버전 이전과 이후의 차이

1.4.1.0 버전은 클라이언트 설치 파일, 설치 파일 다운로드 페이지, 리소스 파일이 모두 dx5 디렉터리 하위에 존재했습니다. 그러나 1.4.1.0 버전 이후부터는 설치 파일과 다운로드 페이지를 하위 client 디렉터리로 따로 묶었으며, 리소스 파일은 하위 res 디렉터리로 이동시켰습니다.

그러므로 1.4.1.0 이전 버전에서 이후 버전으로 업데이트를 할 때에는 디렉터리 관계에 달라지므로 복사를 통한 덮어쓰기 보다, 기존 버전을 완전히 지우고 새 버전을 사용하시길 바랍니다.

dextuploadx5-configuration.js 파일도 일부 속성에 차이가 있기 때문에 기존 파일을 그대로 사용할 수 없으므로 인증키과 각 경로들을 새롭게 설정해주어야 합니다.

멀티 모듈의 경우

멀티 모듈은 클라이언트 설치 작업 없이, 서버에서 내려받은 리소스를 그대로 사용하므로 상위/하위 버전 이동에 제약이나 큰 문제가 없습니다.

그러나 상위 버전은 언제나 하위 버전의 문제점을 수정하기 때문에, 안정적인 서비스 운영을 위해서는 항시 최신 버전을 이용하시길 바랍니다.

상위 IE 모듈을 사용하고자 하는 경우

"업데이트 기본 절차"를 따른 후, IE 모듈을 사용하는 페이지에서 다시 테스트를 해보면 새 버전 교체를 위한 팝업 창이 자동으로 열리게 됩니다.

사용자는 팝업 창에서 새로운 클라이언트 설치 프로그램 다운로드 받을 수 있습니다.

하위 IE 모듈을 사용하고자 하는 경우

"업데이트 기본 절차"를 따른 후, IE 모듈을 사용하는 페이지에서 다시 테스트를 해보면, 상위 버전으로 교체할 때와 달리 팝업 창이 열리지 않습니다.

IE 모듈은 사용자가 사용하고자 하는 제품이 상위 버전일 때만 팝업 창이 뜨도록 구성되었습니다. 그러므로 하위 버전으로 다운그레이드하려면 팝업 창의 도움을 받을 수 없으므로, 버전을 낮추는 일은 신중히 검토해야 합니다.

상위 HD 응용프로그램을 사용하고자 하는 경우

"업데이트 기본 절차"를 따른 후, HD 응용프로그램을 새 버전으로 교체하기 위해 두 가지 방법이 제공됩니다.

사용자의 PC HD 응용프로그램이 설치가 되어 있지 않은 경우에는 사용자가 HD 응용프로그램을 다운로드할 수 있도록 팝업 창이 열립니다. 이와 반대로 설치가 되어 있다면, HD 응용프로그램을 사용하여 다운로드를 하려고 할 때, 자동 업데이트 작업이 실행됩니다.

자동 업데이트 기능은 2.1.2.0 버전부터 지원합니다.

자동 업데이트가 원활하려면 이미 열린 HDM이 있을 경우, 직접 종료시켜야 합니다. (HDM은 윈도우 트레이에 상주합니다.)

자동 업데이트으로 인해 내려 받게 되는 파일은 dextuploadx5-configuration.js 파일의 hd32UpdateURL, hd64UpdateURL 속성에 설정된 주소가 가리키는 대상입니다.
